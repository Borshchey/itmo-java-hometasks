1. Что такое обертки над примитивами?

Обертка — это специальный класс, который хранит внутри себя значение примитива.
Они будут хранить внутри нужные значения примитивов, при этом будут являться настоящими объектами.



2. Когда лучше использовать примитивы, а когда соответствующие классы-обертки?

примтивы:
имеют преимущество в производительности

Обертки:
Позволяют не нарушать принцип “все является объектом”, благодаря чему числа,
символы и булевы значения true/false не выпадают из этой концепции
Расширяют возможности работы с этими значениями, предоставляя удобные методы и поля
Необходимы, когда какой-то метод может работать исключительно с объектами

Джава не позволяет использовать примитивные типы при параметризации (generics), в коллекциях Java или в Reflection API.

3. Что такое автоупаковка?

переменной класса-обертки можно присваивать значение примитивного типа.
Этот процесс называется автоупаковкой (autoboxing).


4. Что такое автораспаковка?

Точно так же переменной примитивного типа можно присваивать объект класса-обертки.
Этот процесс называется автораспаковкой (autounboxing).
